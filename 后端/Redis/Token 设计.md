|维度|Redis|DB (MySQL/PostgreSQL等)|
|:--|:--|:--|
|存储介质|🧠 内存 (RAM)|💾 磁盘 (SSD/HDD) + 内存缓存|
|读写延迟|⚡ 亚毫秒级 (~0.1ms)|🐢 毫秒级 (~1-10ms)|
|并发吞吐|🔥 极高 (10万+ QPS)|📊 中等 (千~万级 QPS)|
|数据持久性|⚠️ 弱 (重启/故障可能丢数据)|✅ 强 (ACID 事务保证)|
|数据结构|KV / Hash / Set / ZSet / Stream|关系表 / JSON / 索引 / 外键|
|过期机制|✅ 原生 TTL (自动清理)|❌ 需手动实现 (定时任务/触发器)|
|查询能力|🔑 仅按 Key 查 (复杂查询受限)|🔍 SQL 全功能 (JOIN/聚合/事务)|
|运维成本|较高 (集群/哨兵/内存监控)|中等 (成熟生态/备份恢复)|
