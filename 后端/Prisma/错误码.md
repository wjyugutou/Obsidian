# Prisma 错误码完整参考

> [!info] 来源
> 官方文档：[Prisma Error Reference](https://www.prisma.io/docs/orm/reference/error-reference)
> 命名规则：`P{系列编号}{错误编号}`

---

## 异常类层级

Prisma Client 抛出 **5 种异常类**，错误码只出现在其中两种：

| 异常类 | 含义 | 有 code？ | 有 meta？ |
|--------|------|----------|----------|
| `PrismaClientKnownRequestError` | 查询引擎返回的**已知**错误 | ✅ `code` | ✅ `meta` |
| `PrismaClientUnknownRequestError` | 查询引擎返回错误但**无对应码** | ❌ | ❌ |
| `PrismaClientValidationError` | **查询参数**校验失败（代码写错） | ❌ | ❌ |
| `PrismaClientInitializationError` | **连接数据库**时失败 | ✅ `errorCode` | ❌ |
| `PrismaClientRustPanicError` | 引擎崩溃，需重启进程 | ❌ | ❌ |

> [!tip] 实践
> 全局异常过滤器里 `@Catch()` 捕获所有异常，但只有 `PrismaClientKnownRequestError` 才有 `.code` 和 `.meta`，按 `code` 分支处理。

> [!abstract] 触发阶段
> 跑 `prisma migrate` 命令时的错误，和运行时 CRUD 无关。

| 码 | 含义 |

| P3010 | 迁移名称太长（>200 字符） |
| P3011 | 迁移无法回滚（从未应用） |
| P3012 | 迁移无法回滚（不在失败状态） |
| P3013 | 不再支持多 provider 数组 |
| **P3014** | 无法创建 shadow database（权限不足） |
| P3015 | 找不到迁移文件 |
| P3016 | 数据库重置的 fallback 方法失败 |
| P3017 | 找不到迁移 |
| **P3018** | 迁移应用失败 |
| P3019 | provider 和 migration_lock.toml 不匹配 |

## NestJS 全局过滤器实现参考

> [!example] 代码示例
> 以下是在 NestJS `AllExceptionsFilter` 中处理 Prisma 错误的核心逻辑：

```typescript
import { Prisma } from '../generated/prisma/client';

// P2002 唯一约束：动态提取冲突字段
const UNIQUE_FIELD_MSG_MAP: Record<string, string> = {
  username: '用户名重复',
  email: '邮箱重复',
};

// MySQL adapter 的 meta.target 可能为 undefined
// 需 fallback 到 driverAdapterError.cause.constraint.index
function extractUniqueField(meta): string | undefined {
  if (meta?.target?.length > 0) return meta.target[0];
  const indexName = meta?.driverAdapterError?.cause?.constraint?.index;
  if (indexName?.includes('username')) return 'username';
  if (indexName?.includes('email')) return 'email';
  return indexName;
}

// 在 filter 的 catch 中分支
if (exception instanceof Prisma.PrismaClientKnownRequestError) {
  switch (exception.code) {
    case 'P2002': // → 409
    case 'P2025': // → 404
    case 'P2000': // → 400
    case 'P2011': // → 400
    case 'P2024': // → 503
    // ...
  }
}
```
